# 코딩 디시플린 (팀 원칙)

모든 코드 작업에 항상 적용한다. 이 파일이 **단일 출처(SSOT)**이며, disciplined-coder 플러그인이
각 프로젝트의 CLAUDE.md에 `@import`로 자동 주입한다. 프로젝트의 사본은 직접 수정하지 말 것
(매 세션 이 정본에서 다시 복사된다).

각 원칙은 짧은 ID로 참조한다(예: `SSOT`). **순서·우선순위는 없다** — 아래는 조회용 **ID 알파벳순 글로서리**이며 위아래 위치에 의미가 없다(그룹·번호 없음).

## 원칙 (ID로 참조 · 무순서 · 알파벳순)
- **`CLEAR-COMM`** — 결론만 던지지 말고 근거·기각한 대안까지 설명한다. 단계를 건너뛰지 않는다. 판단이 필요한 곳에서 모호한 추측 대신 명확히 질문한다.
- **`EXPLICIT`** — 놀라움 최소화. 숨은 동작·매직 금지. 의도는 이름·계약·타입으로 드러낸다.
- **`FAIL-LOUD`** — 조용한 실패 금지(드리프트·계약 위반은 즉시 빨강으로). 구조(불변식·생성된 설정·명확한 계약)가 실수를 막거나 드러내게 한다 — "정확히 기억"에 의존 금지(강건성 > 작성자 정확성).
- **`FOCUSED`** — 한 단위는 한 가지 일. 내부를 몰라도 쓸 수 있게 인터페이스로 의존. 파일 비대화는 분리 신호.
- **`IDEMPOTENT`** — 스크립트·마이그레이션·셋업은 반복 실행해도 안전하게.
- **`MEASURE-FIRST`** — 가정 금지. 환경 차이를 먼저 확인.
- **`NON-DESTRUCTIVE`** — 원본 보존. 필터/dedup/스팸 제거는 삭제가 아니라 분류 표시(mark). 저장 후 처리.
- **`REVERSIBLE`** — 되돌릴 수 있는 결정을 선호. 비가역 결정은 근거를 남긴다.
- **`SECRETS`** — 진짜 비밀은 백엔드 전용. 비밀 아닌 식별자만 클라이언트 노출.
- **`SIMPLE`** — 지금 필요 없는 일반화·추상화 금지, 가장 단순한 동작부터. 코드는 쓰기보다 읽히는 횟수가 많다 — 주변 관례를 따른다.
- **`SSOT`** — 같은 정보는 한 곳에만. 사람이 두 곳을 동기화하게 만들지 않는다(복제 말고 도출).
- **`TDD`** — 실패 테스트 먼저. 실행 증거 없이 "됐다" 금지.

## 환경 관례 (보편 원칙 아님)
- **`DOCKER-ONLY`** — 로컬 실행 금지. 프로덕션과 동일 환경에서.

## 공통 함정 (cross-project gotchas)
- **`.gitignore` 인라인 주석 미지원** — `pattern  # comment`는 안 먹힌다. 주석은 별도 줄로.
- **mock과 실제 클라이언트의 None 차이** — 테스트 더블이 통과해도 실제 SDK는 빈 결과에 `None`을 반환해 운영에서 `AttributeError`가 날 수 있다. → `x or {}`/`x or []` 가드 + 테스트 더블이 프로덕션 계약(빈 결과=None)을 약화시키지 않게 한다.
- **테스트 기대치 매직 넘버 금지** — "PASS=K 달성"처럼 기대 개수를 박지 말 것(체크 가감 때마다 사람이 동기화 = `SSOT` 위반). 계약은 **불변식(예: FAIL=0)**으로 검증하고, 개수는 테스트가 스스로 센다.

## 절차 (원칙과 별개 — 어드바이저·계획 연결)
- **런타임 LLM 절차** — LLM 콜로 끝나는 런타임 기능은 단독 결론으로 마치지 말고, `domain-llm-runtime`(+ `advisor-*`) 스킬에서 리스크에 맞는 검증 레이어를 골라 **제품 코드에 구현**한다.
- **계획 시점 절차** — 설계/계획(brainstorming/writing-plans) 시 **`domains-index`에서 해당 개발 대상의 참고서를 확인**하고 '마땅히 그래야 하는 것'을 명세에 반영한다(명세에 못 담으면 개발 단계에서 고려). 런타임 LLM 기능은 검증 레이어를 포함한다.
- **이슈 로그 생애주기** — 문제를 인지하면 `unsolved_problems.md`에 등록하고, 해결되면 `solved_problems.md`로 옮긴다(항목: 문제 → 원인 → 해결). 발동 시점: **검증/리뷰 작업이 끝날 때 등록**, **테스트가 통과로 바뀔 때 solved로 이동**.
- **단일 작성자 + 보고** — 이 로그 파일들은 **메인 세션(오케스트레이터)만 쓴다**(동시 쓰기 손상 방지). 서브에이전트는 직접 쓰지 말고 발견한 문제를 **리턴값으로 보고**하면 메인이 취합·dedup해 기록한다. 세션 중 최신 항목이 필요한 서브에이전트에는 메인이 dispatch 프롬프트에 관련 항목을 실어 전달한다. (`unsolved_problems.md`의 🔴는 누구도 자율 구현하지 않는다.)
