# 코딩 규율 (general)

프로젝트 고유 사실(특정 피드·프로젝트 ID·특정 버그)은 여기 넣지 말 것 — 그건 각 프로젝트의 `solved_problems.md`에 둔다. 여기는 **어느 프로젝트에서나 참인 일반 규칙**만.

## 원칙
1. **SSOT** — 같은 정보는 한 곳에만. 드롭다운/상수는 유일 출처에서 도출.
2. **복제 말고 도출** — 사람이 두 곳을 동기화하게 만들지 말 것.
3. **Fail-loud** — 드리프트·계약 위반은 테스트가 빨강으로 즉시 드러내야 한다.
4. **강건성 > 작성자 정확성** — 구조가 실수를 막거나 드러내야 한다. 기억/정확 편집에 의존 금지.
5. **TDD + 검증 후 주장** — 실패 테스트 먼저. 실행 증거 없이 "됐다" 금지.
6. **비파괴 우선** — 원본 보존. 필터/dedup은 삭제 아닌 mark. 저장 후 처리.
7. **측정 먼저** — 가정 금지. 환경 차이 먼저 확인.
8. **Docker 전용 개발/테스트** — 로컬 실행 금지. 프로덕션 이미지와 동일 환경에서 테스트.
9. **비밀 분리** — 진짜 비밀은 백엔드 전용. 비밀 아닌 식별자만 클라이언트 노출.

## 프로젝트 간 공통 함정 (general gotchas)
- **[Git Bash 호스트 전용] Docker bind-mount 경로** — Git Bash(Windows)에서 `$(pwd)`/`${PWD}`가 경로 변환으로 망가져 마운트가 stale 이미지로 조용히 폴백(테스트 수가 틀리게 나옴). → `MSYS_NO_PATHCONV=1` + 절대경로(`-v "D:/...:/app"`). **mac/Linux/PowerShell 호스트엔 해당 없음** — 보편 규칙으로 적용하지 말 것.
- **`.gitignore` 인라인 주석 미지원** — `pattern  # comment`는 안 먹힘. 주석은 별도 줄로.
- **mock과 실제 클라이언트의 None 차이** — 테스트 더블이 통과해도 실제 SDK가 빈 결과에 `None`을 반환해 `AttributeError`. → `x or {}` 가드. 테스트 더블이 프로덕션 계약을 약화시키지 않게.
- **"내 환경에서 됨 ≠ 프로덕션에서 됨"** — 데이터센터 IP 차단 등. 외부 의존은 프로덕션 환경에서 측정.

## solved/unsolved 로그 사용 규약
- 작업이 닿는 영역에 대해 프로젝트 `solved_problems.md`를 먼저 확인 → 같은 실수 반복 방지.
- 새 해결은 `문제 → 원인 → 해결` 한 줄로 `solved_problems.md`에 기록.
- `unsolved_problems.md`의 🔴(결정 필요)는 자율 구현 금지. 🟡(방향 정해짐)만 구현 후보.
