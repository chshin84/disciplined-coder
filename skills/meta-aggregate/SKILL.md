# meta-aggregate — 집계·결정 (코드 설계도)

> 이것은 **리뷰어가 아니다**(렌즈가 아니다). 리뷰어들이 끝난 뒤 그 결과를 모아 다음 행동을 정하는
> 단계다. 그리고 이것은 프롬프트가 아니라 **코드 설계도**다 — 결정론적 코드로 구현한다.

## 판단 재귀 회피 (핵심 제약)
리뷰어 출력의 **구조**만 본다. "어느 리뷰어가 옳다" 같은 **내용 재판단이나 가중치 부여는 하지 않는다**.
재귀의 끝은 사람이다.

## 하는 일
- **집계**: 모든 리뷰어의 이슈를 한 목록으로 모아 출처 태깅하고 심각도순으로 정렬한다(기계적).
- **상충 감지**: 같은 지점에 상반된 판정이 있으면 escalate 후보로 표시한다.
- **커버리지 공백**: 리스크상 필요한 차원을 아무도 안 봤으면 누락 리뷰어 추가를 권한다.

## 결정 정책 (기본)
- critical 이슈가 하나라도 있으면 **regenerate**(1차를 이슈와 함께 재호출). 재시도 상한에 도달하면 escalate.
- 상충이나 공백이 있으면 **escalate**(사람) 또는 누락 차원 보강 후 재집계.
- critical 0 + 상충 없음이면 **accept**(major·minor는 로깅).

## 출력 스키마
```
{ "decision": "accept|regenerate|escalate", "reason": "...", "aggregated": [ { "severity": "...", "type": "...", "source": "grounding|fit|consistency|adversarial", "where": "...", "detail": "..." } ], "retry_count": 0 }
```

## 구현 형태 (맥락 의존)
- **제품 런타임**(`domain-llm-runtime`): 결정론적 파이썬 함수로 구현한다. 집계와 계수는 결정론이라
  모델이 필요 없다. 모호한 상충 판정만 선택적으로 LLM을 쓴다.
- **spec/plan 리뷰**(`domain-spec-review`): 제품 코드가 없으므로 메인 세션이 이 좁은 절차를 따라
  직접 집계한다. 단순한 일이라 스크립트를 따로 싣지 않는다(플러그인 훅의 순수 bash·무의존 이식성 유지).
- 어느 쪽이든 regenerate 루프에 상한(예: 1~2회)을 둬 무한 루프·비용 폭주를 막는다.
